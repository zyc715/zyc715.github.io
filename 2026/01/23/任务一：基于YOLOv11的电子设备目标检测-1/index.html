任务一：基于YOLOv11的电子设备目标检测

一、任务背景与目标

本任务为本人第一次目标检测综合实践任务，主要目标是基于 YOLOv11 算法完成电子设备目标检测系统的设计与实现。

检测的电子设备类别包括：

  • 显示器（monitor）
  • 键盘（keyboard）
  • 鼠标（mouse）
  • 手柄（controller）

最终实现一个可在浏览器中运行的检测系统，支持图片与视频的目标检测，并展示检测结果与目标数量统计。

二、数据集制作

2.1 数据采集

数据集图片主要通过以下方式获取：

  1. 从网络搜索相关电子设备图片（不同品牌、角度、背景）
  2. 使用拍摄设备采集真实场景下的电子设备图片

为保证数据多样性，图片包含室内、办公桌、不同光照条件等多种场景。

2.2 数据标注

使用 labelImg 工具对图片进行标注，标注格式选择 YOLO 格式。

标注的目标类别如下：

类别编号 类别名称
0 controller
1 keyboard
2 monitor
3 mouse

每张图片生成一个对应的 .txt 标签文件，记录目标的类别与位置信息。

2.3 数据集划分

数据集总量不少于 200 张图片，每个类别不少于 50 张样本。

按照 8:2 的比例划分数据集：

  • 训练集（train）：80%
  • 验证集（val）：20%

三、YOLOv11 模型训练

3.1 训练环境

  • 操作系统：Windows 10
  • Python 版本：3.x
  • 深度学习框架：PyTorch
  • 目标检测算法：YOLOv11

3.2 模型训练

基于制作的数据集，配置 YOLOv11 的数据配置文件（data.yaml），并启动训练。

训练完成后，成功生成模型权重文件：

runs\detect\train9\weights\best.pt

该权重文件用于后续的目标检测推理。

3.3 模型推理

编写推理代码，加载训练好的模型权重，实现：

  • 图片目标检测
  • 视频目标检测
  • 实时摄像头检测

检测结果以边框和类别标签的形式进行可视化展示。

四、基于 Flask 的网页检测系统

4.1 系统功能设计

使用 Flask 框架开发 Web 应用，主要功能包括：

  1. 图片文件上传并检测
  2. 视频文件上传并逐帧检测
  3. 本地摄像头实时检测
  4. 检测结果展示
  5. 目标数量统计显示

4.2 界面展示

网页界面包含以下模块：

  • 文件上传按钮
  • 检测结果展示区域
  • 状态信息与目标数量统计区域

检测完成后，网页实时显示带标注框的图片或视频画面。

五、实验结果与分析

实验结果表明，训练后的 YOLOv11 模型能够较为准确地识别显示器、键盘、鼠标和手柄等电子设备目标。

在光照良好、目标清晰的情况下，检测效果较好；在复杂背景或遮挡情况下，部分目标存在漏检现象，后续可通过增加数据集和训练轮数进行优化。

六、总结与展望

通过本次任务，完成了从数据集制作、模型训练到 Web 系统部署的完整目标检测流程。

本任务不仅加深了本人对 YOLO 系列目标检测算法的理解，也提升了对 Flask Web 开发和工程实践的能力。

后续工作可考虑：

  • 增加数据集规模
  • 支持多类别筛选检测
  • 引入多线程提升视频检测性能
  • 支持检测结果导出功能
